NAFIPS 2026 Hack-a-thon

This year in addition to the normal XFC competition, we'll be holding an additional competition at the 2026 NAFIPS conference in El Paso, Texas, USA. This competition will be a "Hack-a-thon" where student competitors will be given tutorials and guided work sessions during the conference and then they will go off on their own to create their AI agents to play Kessler in a live competition! Students will develop skills including ML/RL with gradient-free methods, Python object oriented programming, and project management!

Registration is now closed for the Kessler Hackathon, but if interested can still use the development guide and files to work on your own. Link will be posted for the guide and files soon.

Competition Overview

The Kessler AI competition at NAFIPS 2026 is a competition for high school and university level students to develop an AI agent to play the Python game Kessler which is also on PyPi. Kessler is an open-source Python game based on the classic arcade game "Asteroids" where ships shoot/destroy asteroids to gain score. Kessler is also adversarial in that multile agents can be in the same game simultaneously. The AI agents created will be evaluated in a playoff style competition in a session at FUZZ-IEEE where teams will directly compete against each other in the game for the most points. The playoff will be single elimination with an additional 3rd place game and play-in games/byes if necessary based on the number of entries. Bracket seeding will be determined by evaluation of teams' agents against baseline agents provided by the organizers.

The overall motivation for having the competition is to increase student's skills with respect to coding, repository management, machine learning, and project managment. Also we want to both showcase the applicability of Fuzzy towards problems like this, while also getting more exposure for students in actual application development in a fun and rewarding way.

This competition is inspired by, and an offshoot of, the Explainable Fuzzy Challenge (XFC) which has been run each year since the spring of 2021. This competition is associated with the North American Fuzzy Information Processing Society conference and is sponsored each year by Thales. The focus of this competition is even more broad in scope than the NAFIPS Kessler competition as it includes an explainable AI component. XFC has had good participation from the NAFIPS community and a number of universities and was directly responsible for the desire to have a hack-a-thon version live at the conference. As the focus of XFC is more of an educational experience as opposed to pure competition (as is intended for this version), there are a number of recorded seminars on topics relevant for the competition such as Genetic Algorithms, Fuzzy Logic and Fuzzy Inference Systems, Reinforcement Learning, and repository and code management practices. These are available on the XFC YouTube channel.

Team Info

There are a few limitations on teams. First is that teams must be comprised of 1-6 high school/university level students. Students of any major/focus are welcome. There are a few pre-requisites however as the competition requires competency in repository usage, Python coding, object-oriented programming, and implementation of machine learning algorithms.

Rules

There are several rules associated with the competition. Some of these are somewhat flexible and are denoted as such, but most are to be adhered to in order to make the competition go smoothly.

Rule 1
(Somewhat Flexible) The AI agents developed must utilize Fuzzy logic in some way. Fuzzy Inference Systems, Fuzzy Trees, NeuroFuzzy architectures, etc. are all allowed and encouraged. Other architectures and methods may be allowed on a case-by-case basis but require approval from the organizers. This rule exists to encourage usage and application of Fuzzy in relevant (and fun!) problems.
Rule 2
Any machine learning method (or none) is allowed including reinforcement learning, evolutionary/genetic algorithms, and expert knowledge encoding. Note: Kessler currently only supports delayed rewards (total reward given at end of episode) rather than on each timestep. This can be changed as Kessler is open source, but for this competition has not been implemented.
Rule 3
Teams must maintain their code in a file structure to be determined by the organizers. They may hold their code in a repository or only locally, but must be able to be delivered in a single compressed file (zip, rar, etc.) with the requisite structure for easy integration for the competition.
Rule 4
The solutions must be able to interface with the game in Python as described in the Controller class in Kessler, but there are no other prescriptions for development style or coding paradigms. E.g. one team may decide to code in pure Python while another may compile some heavy calculations associated with their controller using C++, Cython, mypyc, etc. but then have a Python wrapper to interface with Kessler. Both of these are allowed with the stipulation that all competition teams must resolve all potential compilation issues for target hardware with the organizers before the final agent delivery deadline.
